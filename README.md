# AI Legal Document Processing Pipeline

This project is a pipeline for analyzing, indexing, drafting, and continuously optimizing legal document processing using Retrieval-Augmented Generation (RAG) and Groq (Llama 3).

---

Overview

This project provides an end-to-end architecture designed to ingest messy legal-style documents, extract their structured content, index them into a vector database, and use AI to generate grounded analytical drafts. It also has a learning loop that captures human-operator formatting edits to continuously optimize future drafts.

# Architecture Overview

Flowchart
https://excalidraw.com/#json=pFmD3wKpI2N7aBotLqjRP,DXfkBFqq-GGIp673Ul0XPA

Details
The pipeline is split into four primary segments, mapping directly to the assessment requirements.

Segment 1: Document Processing
Parses text from PDF files. If a document appears scanned (low text volume), it triggers an OCR fallback using Tesseract.
use Groq (Llama 3) to structure the raw text into a JSON schema (identifying Document Type , Parties Date, etc.) and chunks it into normal readable paragraphs, ensuring the output is perfectly prepared for next upcoming stages.

Segment 2: Grounded Retrieval
Generates vector embeddings from the text chunks using local sentence-transformers (all-MiniLM-L6-v2).
Indexes the vectors into a local ChromaDB instance with explicit metadata, which makes system to get only the most relevant evidence chunks.

Segment 3: Draft Generation
Retrieves the most relevant evidence chunks from ChromaDB based on the provided drafting task/prompts (which u can see by clicking on prompts button at the topright of the page).
Prompts Llama 3 to generate a professional, structured first-pass draft using only the provided factual evidence chunks.
Demands that every factual claim is appended with its source ID to ensure traceability and grounding.

Segment 4: Improvement from Operator Edits
captures operator edits on the initial draft.
Extracts reusable style and structural rules by comparing the initial draft against the operator's version.
Shows the learned rules as alert panel and improved draft on draftwindow.
Saves these preferences persistently (data/operator_preferences.json) to dynamically guide future generation prompts, establishing a continuous improvement loop.

# Evaluation Approach and Results

We evaluated the system against the 4 steps.

Document Processing: Tested with clean scanned PDFs. The fallback mechanism correctly identified missing text and invoked OCR to extract readable tokens.

Retrieval and Grounding: We purposefully tested requests for dollar amounts and dates. The system correctly retrieved the exact sentences from the chunks, and the generator strictly appended source IDs (e.g., [Source: doc_pdf_001_chunk_0]) to the output, proving no hallucination occurred.

Draft Quality: The initial draft is consistently output as a structured Markdown summary, highlighting financial obligations, rent, and rules as requested, directly based on the ingested contract.

Improvement Loop: We evaluated the learning mechanism by injecting a simulated edit that changed formatting (e.g., bolding "$4,200" and changing a header). The system successfully extracted this rule ("Format financial figures in bold") and applied it automatically to the final optimized draft.

# Option 1: Run with Docker Compose
We have provided a ready-to-use Docker environment that perfectly mimics the production Hugging Face deployment without requiring any local dependencies.

1. Ensure Docker Desktop is installed and running.
2. Create a .env file in the root directory and add your Groq API Key:
GROQ_API_KEY=your_api_key_here
3. Start the application:
docker compose up --build

4. Open your browser and navigate to http://localhost:8000
5. You can now test the entire assignment.



Option 2: Manual Python Setup
If you prefer running without Docker, you can set up the environment and run the FastAPI server manually:

1. Prerequisites
Python 3.8+

2. Install Dependencies
pip install -r requirements.txt

3.  Environment
Create a .env file in the root directory and add your Groq API Key:
you can get it from https://console.groq.com/

python main.py

5. Go on 8000 and enjoy.
